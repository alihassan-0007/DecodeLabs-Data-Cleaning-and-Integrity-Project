# DecodeLabs Data Cleaning and Integrity Project

## Project Overview

This project focuses on improving data quality through a structured data cleaning and validation process. The objective was to transform a raw dataset into a clean, consistent, and analysis-ready dataset by addressing common data quality issues.

## Objectives

* Identify and handle missing/null values.
* Remove duplicate records.
* Standardize data formats.
* Validate data integrity and consistency.
* Prepare the dataset for further analysis and reporting.

## Tools Used

* Microsoft Excel
* Power Query
* Conditional Formatting
* Excel Functions (COUNTA,Go To Special)

## Data Cleaning Process

### 1. Handling Missing Values

Missing and null values were identified and processed using Power Query to improve data completeness and consistency.

### 2. Removing Duplicates

Duplicate records were detected and removed using Excel's Remove Duplicates feature.

### 3. Data Formatting

Date, numeric, and text fields were standardized to ensure accurate calculations and analysis.

### 4. Data Validation

The dataset was reviewed for inconsistencies and corrected where necessary to improve overall data quality.

## Data Quality Verification

### Record Validation

The COUNTA function was used to verify the total number of records after the cleaning process.

### Conditional Formatting

Conditional formatting was applied to identify anomalies, blank cells, duplicate values, and potential outliers.

### Format Consistency Check

Data type mismatches were identified and corrected to ensure consistency across the dataset.

## Results

* Successfully transformed raw data into a clean and structured dataset.
* Improved data accuracy, consistency, and reliability.
* Enhanced data integrity through systematic validation procedures.
* Prepared the dataset for exploratory data analysis and business reporting.

## Conclusion

Through the use of Power Query, data validation techniques, conditional formatting, and Excel-based quality checks, the dataset was successfully cleaned, standardized, and optimized for analysis. Missing values, duplicate records, formatting inconsistencies, and potential data quality issues were systematically identified and resolved. The final dataset is accurate, reliable, and ready for further analysis and data-driven decision-making.
